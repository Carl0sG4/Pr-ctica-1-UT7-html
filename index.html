Introducción al Escenario

Este manual documenta el despliegue técnico de la oficina técnica de la empresa tecnológica "OpenCloud Solutions". Se ha configurado una infraestructura segura basada en Linux utilizando VirtualBox, que incluye un controlador de dominio, servidor de ficheros departamental, servidor web, y medidas de seguridad perimetral.

Fase A: Acceso a Recursos y Dominios

A.1. Configuración de Red en VirtualBox

Para garantizar el aislamiento y la conectividad a Internet de nuestro servidor, necesitamos configurarle dos interfaces de red físicas virtuales. El cliente, por otro lado, solo debe tener acceso a la red interna corporativa.

A.2. Configuración de Red Estática (Netplan)

En Ubuntu Server, asignamos la IP fija requerida (192.168.100.1) a la segunda interfaz del servidor mediante Netplan.

bash - ubuntu@srv-admin
# Accedemos a editar el archivo de configuración de Netplan
sudo nano /etc/netplan/50-cloud-init.yaml

Añadimos la configuración para enp0s8 respetando la sintaxis YAML:

50-cloud-init.yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
    enp0s8:
      addresses:
      - 192.168.100.1/24
# Aplicamos los cambios en la red
sudo netplan apply

A.3. Configuración de Red en el Cliente Mint

# Prueba de conectividad ICMP desde la terminal de Mint
ping -c 4 192.168.100.1

A.4. Montaje Permanente de Datos

# 1. Formateamos el disco al sistema de archivos EXT4
sudo mkfs.ext4 /dev/sdb

# 2. Creamos la carpeta donde montaremos el disco
sudo mkdir -p /mnt/datos_empresa

# 3. Lo montamos temporalmente
sudo mount /dev/sdb /mnt/datos_empresa

# 4. Lo hacemos persistente en fstab
sudo nano /etc/fstab

A.5. Controlador de Dominio (Samba4 AD DC)

sudo cp /var/lib/samba/private/krb5.conf /etc/krb5.conf
sudo systemctl unmask samba-ad-dc
sudo systemctl enable samba-ad-dc
sudo systemctl start samba-ad-dc

# Comprobamos estado del dominio
sudo samba-tool domain level show
